1. Что нужно настроить перед запуском (обязательно)
Запусти Chrome вручную с remote debugging через терминал.
Это нужно один раз (или каждый раз при запуске рассылки), Chrome должен быть запущен до старта скрипта.

Пример запуска Chrome for macOS:
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --remote-debugging-port=9222 \
  --user-data-dir="$HOME/chrome-profile-tg1"

Если хочешь использовать прокси, установи его при запуске Chrome (system or Chrome launch flags). Скрипт НЕ меняет прокси на лету в режиме Remote Debugging (так надежнее).


2. Открой в этом Chrome вручную https://web.telegram.org/k/ и выполни вход в telegram (QR/сессия) — убедись, что профиль уже авторизован. Скрипт будет работать с уже авторизованной сессией.
Проверь, что chromedriver версии совместим с установленным Chrome (в большинстве случаев при attach это не критично, но лучше иметь совместимую версию). 

3. Установи selenium или сразу файл requirements.txt

4. В files_json/config.json укажи максимальное кол-во отправлений, чтобы телеграм не посчитал это как спам.
Также в этом файле укажи текст сообщения для рассылок

5. В files_json/contact.json укажи имя клиента для обработки ошибок, если будут какие-то проблемы.
Вставь ссылку телеграма на чат с человеком.

6. В files_json/progress.json проеверь что last_index = 0, это номер человека на котором скрипт остановится, если произойдет какой-то сбой. И начнет он тогда с человека, на котором остановился.

7. Перейди в папку в которой у тебя скачан данный проект и запусти основной файл main.py - рассылка запустится
